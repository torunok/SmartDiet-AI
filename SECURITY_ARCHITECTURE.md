## ğŸ“ Key Takeaways

1. **Never put API keys in frontend code**
   - Frontend is visible to users
   - Any secret here is compromised

2. **Use backend for API calls**
   - Backend is private to your server
   - Only your server can access .env

3. **Use environment variables**
   - .env files for secrets
   - .env.example for templates
   - Never commit .env to git

4. **Implement CORS properly**
   - Control which domains can access API
   - Prevent CSRF attacks

5. **Validate and log requests**
   - Server-side validation is important
   - Logging helps with debugging and security
